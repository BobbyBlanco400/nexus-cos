# PR & AMPLIFICATION STRATEGY (OPTIONAL)

**STATUS:** Optional but powerful if deployed correctly  
**PRINCIPLE:** PR amplifies FINALITY, not FEATURES  
**RISK:** Done wrong, PR kills the mystery

---

## CORE PR RULE

> **"PR is about the PROCESS, not the PRODUCT."**

We're not launching a platform.  
We're documenting a historical funding moment.

**The story is:** "How a platform funded infrastructure before anyone knew what it was."

**NOT:** "Here's a new platform with cool features."

```

```

## PR DEPLOYMENT STRATEGY

### Timing:

**Option A: Pre-Close PR (Dec 28-30)**
- Amplifies campaign during final days
- Drives awareness to Genesis Pass deadline
- Creates external validation
- Risk: Brings skeptics and low-quality attention

**Option B: Post-Close PR (Jan 2-5)**
- Documents the completed event
- No risk of diluting message (pass is closed)
- Positions as case study
- Benefit: "Look what just happened"

**RECOMMENDATION: Post-Close PR (Option B)**

Why:
- ✅ Protects campaign purity during execution
- ✅ Allows storytelling without selling
- ✅ Builds authority for public launch
- ✅ No pressure to convert skeptical press

## RISK MITIGATION

### Potential PR Risks:

1. **Risk: Media demands feature explanations**
   - Mitigation: Politely refuse. Stick to process story.

2. **Risk: Coverage frames as "scam" or "sketchy"**
   - Mitigation: Lead with real costs ($497 THIIO). Show receipts.

3. **Risk: Public asks for "reopening"**
   - Mitigation: Restate: "No reopening. Ever. Finality is the point."

4. **Risk: Competitors copy strategy**
   - Mitigation: None needed. Execution is the moat, not the idea.
